fix: mirror NVS semantics exactly for blob size queries

Addresses feedback to align config_store_get_blob() with NVS behavior.

Issues fixed:
1. Blob size query not supported (out == NULL rejected)
2. Required size not returned on ESP_ERR_NVS_INVALID_LENGTH

Changes:
- Allow out == NULL for size-only queries (mirrors nvs_get_blob)
- Write required size to *out_len when buffer too small
- Updated validation: only require out_len != NULL
- Callers can now discover required size without upfront buffer

Behavior:
- config_store_get_blob(ns, key, NULL, &size) -> Returns size in *size
- config_store_get_blob(ns, key, small_buf, &size) -> Returns ESP_ERR_NVS_INVALID_LENGTH
  and writes required size to *size
- config_store_get_blob(ns, key, buf, &size) -> Reads blob and updates *size with actual

Testing:
- Added test_config_store_blob_size_query test case
- Updated blob_buffer_too_small test to verify *out_len is updated
- Added <stdlib.h> for malloc/free in tests

- components/config_store/config_store.c
- components/config_store/test/test_config_store.c
