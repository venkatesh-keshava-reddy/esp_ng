feat: implement config_store NVS abstraction layer

Implements schema-agnostic typed helpers for NVS storage operations.

Features:
- NVS initialization with automatic no-free-pages handling
- Typed operations: string, u32, blob
- Namespace/key length validation (15 char limit)
- Proper error propagation (ESP_ERR_NVS_NOT_FOUND for missing keys)
- Buffer size validation with ESP_ERR_NVS_INVALID_LENGTH
- Null-termination for string operations
- Erase key functionality
- Set-if-missing helpers for idempotent defaults

Implementation details:
- Open/close NVS handle per operation (thread-safe, no global state)
- Validates namespace and key lengths against NVS limits
- Comprehensive error handling with logging
- Follows ESP-IDF NVS semantics exactly

API:
- config_store_init() - Initialize NVS flash (handles erase/retry)
- config_store_get_str/set_str() - String operations
- config_store_get_u32/set_u32() - u32 operations
- config_store_get_blob/set_blob() - Binary data operations
- config_store_erase_key() - Delete key
- config_store_set_if_missing_str/u32() - Conditional set

Testing:
- Comprehensive unit tests for all operations
- Roundtrip tests for each data type
- Buffer size validation tests
- Namespace/key validation tests
- Set-if-missing behavior tests
- Null-termination verification
